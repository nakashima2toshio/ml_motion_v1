# ⚙️ 本番/最適化画面 操作マニュアル

**画面**: 左ナビ「本番/最適化」（`app/views/production.py`）。

## 目的
- **バッチ推論**：ディレクトリ内の動画を一括処理。
- **モデル変換・量子化**：ONNX / CoreML / TorchScript / TensorRT へ変換、FP16/INT8 量子化。
- **レイテンシ計測**（説明）／ **Model Registry からの取得**（URI 確認）。

## 事前準備
- バッチ：入力ディレクトリに動画を配置（例 `data/incoming/*.mp4`）。
- 変換：変換元の重みパス（例 `yolo11s.pt`、学習済みなら `best.pt`）。CoreML は Mac 推奨。

## 操作手順

### A. バッチ推論
1. **入力ディレクトリ**に動画を置く（`data/incoming` など）。
2. **📁 入力ディレクトリを確認** で対象が見えるかチェック。
3. モデル・信頼度・フレーム間引きを設定し **▶ バッチ実行**。
4. 進捗（`処理中… n/total ファイル`）後、`成功/失敗/総検出` とマニフェスト表を表示。注釈付き動画は**出力ディレクトリ**に保存。

### B. モデル変換・量子化
1. **重みパス**・**書式**（onnx/coreml/torchscript/engine）・**量子化**（FP32/FP16/INT8）を設定。
2. **🛠 変換を実行**（初回は時間がかかる）。完了で出力ファイルパスを表示。

### C. Model Registry からの取得
- モデル名とステージを選ぶと `models:/<name>/<Stage>` の **URI** が表示される。実運用ではこの URI から取得してバッチ/リアルタイムのモデルを差し替える。

## 出力・結果の見方
- バッチ：マニフェスト表（input / output / frames / detections / status）＋ 成功・失敗・総検出数。
- 変換：出力パス（例 `yolo11s.onnx`）。
- レイテンシ計測：`benchmark_processor()` で mean/p50/p95/fps を計測（**実計測は実機 M2 Mac 上で**）。変換前後の比較に使う。

## つまずきやすい点（トラブルシュート）
| 症状 | 対処 |
|---|---|
| 「動画が見つかりません」 | 入力ディレクトリのパスと動画拡張子（mp4/mov/avi/mkv）を確認 |
| バッチが失敗（一部/全部） | マニフェストの status（`error: …`）を確認。1件失敗でも他は継続する |
| 変換が失敗 | 重みパス、書式の対応（CoreML は Mac）、初回はネット/依存を確認 |
| INT8 量子化で精度低下 | 用途に応じて FP16 / FP32 を選ぶ |
