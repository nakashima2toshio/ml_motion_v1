# 🏷 アノテーションQA画面 操作マニュアル

**画面**: 左ナビ「アノテーションQA」（`app/views/annotation_qa.py`）。

## 目的
フレーム画像と**提案ラベル**を Claude Vision に渡し、bbox/ラベルの**妥当性レビュー**（妥当 / 要修正と理由）を得る。

## 事前準備
- `.env` に `ANTHROPIC_API_KEY` を設定（このページは Claude API を使う）。
- 使用モデルは画面上部に表示（既定 `claude-opus-4-8`、`.env` の `ANTHROPIC_MODEL` で変更可）。
- レビューしたいフレーム画像（`.jpg` / `.jpeg` / `.png` / `.webp`）。解析画面の動画から切り出した1枚などを使う。

## 画面の構成
| 位置 | 項目 | 種類 | 既定 | 説明 |
|---|---|---|---|---|
| 上部 | モデル表示 | 表示 | claude-opus-4-8 | 使用する Claude モデル |
| 左 | フレーム画像をアップロード | ファイル | — | jpg/jpeg/png/webp。アップ後にプレビュー |
| 左 | 提案ラベル（カンマ区切り） | 入力 | person,car | 画像に付けたラベル候補 |
| 左 | 🔍 Claude でレビュー | ボタン | — | 画像アップ後に押下可 |
| 右 | 検出された問題 | 表示 | — | Claude の判定（Markdown） |

## 操作手順
1. **フレーム画像をアップロード**（左にプレビュー表示）。
2. **提案ラベル**をカンマ区切りで入力（例 `person,car,bicycle`）。
3. **🔍 Claude でレビュー** を押す。
4. 右ペインに、各ラベルの**妥当/要修正の判定と理由**が Markdown で表示される。

## 出力・結果の見方
- 各提案ラベルについて「妥当」または「要修正（正しいと思われるラベル＋理由）」が返る。
- bbox が対象を内包しきれていない、ラベル不一致（例：車→自転車）などの指摘に使える。

## つまずきやすい点（トラブルシュート）
| 症状 | 対処 |
|---|---|
| ボタンが押せない | 先に画像をアップロードする |
| 「レビューに失敗しました」 | `.env` の `ANTHROPIC_API_KEY` を確認（未設定/無効） |
| 別モデルで使いたい | `.env` の `ANTHROPIC_MODEL` を変更（例 軽量モデル） |
| 画像形式エラー | jpg/jpeg/png/webp を使用 |
